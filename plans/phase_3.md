# Milestone Plan

**General Information**

Develop a temporal graph neural network to predict token prices leveraging the data in parquet files generated by phase_0_plan.md

**Milestone 1: Foundational Graph Construction**

- **Goal:** Build the initial static, bi-partite graph structure using PyTorch Geometric.
- **Tasks:**
  1.  Define a `HeteroData` schema for the graph with two node types: `Token` and `Pool`.
  2.  Assign unique integer IDs to every token and pool entity.
  3.  Construct the static graph topology by creating undirected edges between tokens and pools based on which tokens are paired in each pool (e.g., if Pool X pairs Token A with Token B, create edges `Token A <-> Pool X` and `Token B <-> Pool X`).
  4.  Add static pool features: fee tier, pool address, and the two token addresses for each pool.
  5.  Initialize token node features with token address and symbol.
- **Validation:** We will check the total number of Token nodes, Pool nodes, and edges. We will verify that each pool has exactly 2 edges connecting to its constituent tokens. We will manually inspect several pools to ensure correct token-pool connectivity.

**Milestone 2: Temporal Feature Engineering**

- **Goal:** Create a temporal graph by adding timestamped swap events as messages/edges and engineering node features that evolve with each swap.
- **Tasks:**
  1.  Create a chronologically ordered sequence of swap events from Milestone 2, where each swap event contains:
      - Timestamp
      - Pool ID (which pool the swap occurred in)
      - Token_in ID and amount_in
      - Token_out ID and amount_out
      - Derived features: implied price of each token in WETH at swap time (from Milestone 3), swap volume in WETH
  2.  Structure these swaps as temporal messages/edges that flow through the graph: `Token_in -> Pool -> Token_out`
  3.  Initialize and update token node features chronologically as swaps are processed:
      - **Token Nodes:** Current WETH price, rolling price volatility (e.g., std dev over recent window), cumulative trading volume, time since last swap
  4.  Initialize and update pool node features chronologically as swaps are processed:
      - **Pool Nodes:** Total liquidity in WETH (if available from events), cumulative swap volume in WETH, number of swaps, time since last swap
  5.  Store the graph structure and the complete temporal swap event sequence in a format suitable for temporal GNN training (e.g., PyG `TemporalData` or custom format with static graph + event list)
- **Validation:** We will sample several tokens and pools and trace their feature evolution over time. We will verify that token prices match those from Milestone 3. We will create visualizations showing how token prices and pool activity evolve in response to swap events. We will verify the temporal ordering is correct and that all swaps reference valid token and pool IDs.
